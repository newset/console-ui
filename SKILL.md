# Agent Console UI · 智能体控制台设计体系

你是一位专做 B 端 AI 工作台的 UI 设计师。你的产出是**能直接打开的 HTML 页面**,风格锁定为「AI 智能体运营控制台」——现代中文 B2B SaaS 里那种信息密度高、以对话为中心的操作台(AI 投放副驾、WhatsApp 线索 CRM、AI 客服工作台)。

一句话概括这个风格:**安静的浅灰画布、白色圆角面板、全页只有一个高饱和蓝、大量语义化小标签、等宽字体的数字、以及位于一切中心的聊天线程。**

## 使用前提

适用:后台/管理台/运营工作台、CRM/收件箱/客服台、AI copilot 控制台、数据看板、设置页——任何"操作员整天盯着"的 B 端界面。

不适用:营销落地页、C 端内容门户、移动端 App 原型、需要强品牌个性的官网。这些场景请不要套用本体系(会显得像后台)。

## 十条铁律(优先级最高,逐条自查)

比任何单个组件都重要。做完页面用这十条过一遍:

1. **画布与面板对比要克制。** 页面底 `--ac-bg`(#F4F6F9),面板纯白 + 1px `--ac-border` + 几乎不可见的阴影。禁止重阴影、禁止灰卡叠灰底。
2. **全页只有一个响亮的颜色。** 饱和蓝 `--ac-primary` 只用于:发出的聊天气泡、每个面板至多一个主按钮、激活态 pill/chip、链接、小节标题的竖条。其余一律中性色或粉彩。
3. **数字必须等宽。** 金额(`$90.00`)、区间(`675-1800`)、电话、ID、计数、排期时间——全部加 `.ac-num`。这是本风格辨识度最高的一条。
4. **标签小、方、粉彩、语义固定。** 4px 圆角、12px 字号、同色系"浅底深字"。绿=质量/成功,红=人工/紧急,蓝=AI/信息,金=高价值,灰=中性。筛选 pill 则是全圆角,别混淆。
5. **气泡承载双语。** 发出气泡实心主色白字;双语时译文在同气泡内下方,72% 白、略小字号。接收气泡白底描边。元信息(发送者·时间·已读)在气泡外,12px 灰。
6. **密度高但不挤。** 正文 13px、次要 12px,卡内 8px 垂直节奏,卡片内边距 12–16px。大字号只属于等宽指标(20–24px)。
7. **小节标题带蓝色竖条。**(`.ac-accent-heading`)3×14px 圆角竖条,用于 AI 回答内的标题和面板小节标题。
8. **右栏是"事实卡"。** 详情/摘要面板 = 标题行 → 标签行 → 键值行(灰label定宽左、深值右)→ 操作按钮的固定叠放。数值一律等宽。
9. **状态用圆点和横幅表达。** 状态 = 6px 彩点 + 文字(琥珀`已暂停`、绿`进行中`);跨面板状态 = 全宽浅色横幅带图标(琥珀=警示/接管,绿=健康/窗口内)。
10. **中文优先的微文案,数据保留英文。** 按钮标签用 2–6 字中文(`看数据`、`恢复投放`、`标记丢单`);实体名、代码、指标保留拉丁字符(`Geely_Starshine6_CTW_B2B`)。这个混排就是气质本身。用户要纯英文界面时,保持同等简练(`Resume`、`Mark lost`)。

## 标准流程

1. 判定布局原型(路由表),从最接近的 example 复制整页骨架——**不要从空白开始写**,这个体系的价值就在于骨架已经对了。
2. 引入 `assets/agent-console.css`(相对路径或内联),根元素挂 `ac-app`。
3. 用 `references/components.md` 的配方填充各栏,改内容不改结构;每个面板至多一个 `.is-primary` 按钮。
4. 变体需求用 data 属性解决:`data-ac-theme="dark"` / `data-ac-accent="green|violet|orange"` / `data-ac-density="compact"`,可组合。**变体禁止手改任何色值**——令牌会级联。
5. 用十条铁律逐条自查,重点:#2(蓝是否泛滥)、#3(数字是否等宽)、#4(标签语义是否错色)。
6. 在浏览器里打开与 examples 并排对比:应该像同一个产品家族的两个页面。

## 禁忌清单(出现任何一条即返工)

- 面板或按钮上出现渐变(仅素材卡占位图允许)。
- 纯黑 #000 文字(正文是 `--ac-text` #1F2329)。
- 蓝上叠蓝:主按钮出现在蓝色横幅或气泡内。
- 卡片大圆角(>16px)或胶囊形卡片;胶囊只属于 pill/chip。
- 用阴影区分层级(应该用描边和底色深浅)。
- 面板内容居中(本体系一律左对齐、顶部锚定)。
- 生产界面用 emoji 当图标(应为 16–18px、1.5px 描边的内联 SVG;demo 里少量 emoji 可接受)。
- 为了"个性"引入第二个饱和色(见铁律 #2)。
